Manual / Introduction

Welcome to WebBuilder

WebBuilder is a web platform that converts any public API into a deployable website using generative AI to analyze the data, propose an architecture and write the code.

What can you do with it?

Skip the repetitive work of turning a dataset into a usable site. Paste a URL, supervise the AI's proposal, and download a real Django project ready to run.

  • Generate blogs, catalogs, dashboards, portfolios — whatever fits your data.
  • Customize the AI's plan with natural language if you don't like the default.
  • Get models, views, templates with Tailwind, and a Dockerfile in one ZIP.
  • Save every analysis to your personal history for later review.
This is a Final Degree Project. The platform is in active development — expect rough edges. Your feedback helps.

Manual / Usage / Paste your URL

Step 1 — Paste your API URL

The first step is to provide a public API endpoint that returns JSON or XML data.

URL requirements

  • Must be publicly accessible (no auth headers supported yet).
  • Must return JSON (application/json) or XML (application/xml).
  • Should ideally return a list/array of structured records.

Examples that work well

# Copy any of these and paste directly into WebBuilder
          https://pokeapi.co/api/v2/pokemon?limit=20
          https://jsonplaceholder.typicode.com/posts
          https://restcountries.com/v3.1/all
          https://fakestoreapi.com/products
Tip: The more structured the data, the better the AI proposal. APIs with consistent fields across records work best.
Manual / Usage / Review the plan

Step 2 — Review the AI plan

The LLM analyzes the dataset and proposes a complete plan. Your job is to review it and decide.

What's in the proposal

  • Site type — blog, catalog, portfolio, dashboard, etc.
  • Relevant fields — what to show and what to ignore.
  • Human-readable labels — clean names for each field.
  • Data preview — first few normalized records.

Edit the schema before generating

If the proposal doesn't match what you want, use the editor to adjust fields, labels and site type — or write a natural language instruction to regenerate the plan entirely.

# Example custom prompts to guide the AI
          "Make it a minimalist portfolio, hide the price field."
          "Show this as a calendar of events grouped by month."
          "Highlight the rating field prominently in each card."
Manual / Usage / Generate project

Step 3 — Generate the project

Once the plan is approved, WebBuilder generates the complete source code.

What gets generated

  • models.py — Django models adapted to your data.
  • views.py & urls.py — routes and view logic.
  • templates/ — HTML with Tailwind CSS, responsive.
  • load_data — management command that fetches and loads the data.
  • Dockerfile & entrypoint.sh — ready for deployment.
Generation takes 20–60 seconds depending on the LLM provider. You'll receive an email notification when it's done.
Manual / Usage / Download & deploy

Step 4 — Download and deploy

Download the ZIP, unzip it, and you have a full Django project ready to run.

Run it locally

$ unzip my-site.zip && cd my-site
$ pip install -r requirements.txt
$ python manage.py migrate
$ python manage.py load_data
$ python manage.py runserver

Or with Docker

$ docker build -t my-site .
$ docker run -p 8000:8000 my-site

Visit http://localhost:8000 and your generated site is live.

Manual / More / Configuration

Configuration

WebBuilder reads its LLM configuration from a .env file. Change provider with three variables.

.env example

LLM_BASE_URL=https://api.groq.com/openai/v1
LLM_API_KEY=your_api_key
LLM_MODEL=llama-3.3-70b-versatile

Supported providers

Any provider implementing the OpenAI /chat/completions format. Tested with:

  • Groq — fast inference, free tier.
  • OpenRouter — broad model catalog with one API.
  • OpenAI — original endpoint, GPT-4 models.
  • Local models — via Ollama or LM Studio with OpenAI-compatible API.
Never commit your .env file to git. Use .env.example as a template instead.
Manual / More / Common errors

Common errors & fixes

Something went wrong? Here are the most frequent issues and how to solve them.

PARSE ERRORThe API URL returns an error or empty data

Check that the URL is publicly accessible and returns JSON or XML. Some APIs require an API key or specific query params. Try opening the URL in your browser first.

LLM ERRORThe AI plan generation fails or times out

Usually a provider issue — check your LLM_API_KEY is valid and the model name is correct. Try switching to Groq (fastest) or OpenRouter if your primary provider is down.

SYNTAX ERRORThe generated code has a syntax error

This can happen with smaller models on complex datasets. Use the regenerate button with a corrective prompt like "Fix any syntax errors and keep the same structure." GPT-4 or Claude produce fewer of these.

DOCKER ERRORThe Docker build fails after downloading the ZIP

Ensure Docker is running locally. Check the requirements.txt inside the ZIP — sometimes the LLM adds a package that doesn't exist. Remove any unknown package and rebuild.

SCHEMA ERRORThe AI proposes wrong fields or misidentifies the site type

Use the regenerate prompt to guide the AI. For example: "This is a list of products. Use name, price and category. Generate a catalog layout." The more specific, the better.

If you find an error not listed here, check the analysis history — the full error message is stored there for each failed generation.
Manual / More / FAQ

Frequently asked questions

Common questions about WebBuilder.

Does WebBuilder support private APIs with authentication?
Not yet. The first version only supports public endpoints. Auth support (headers, OAuth) is on the roadmap.
Can I edit the generated code?
Absolutely. The generated project is a regular Django project — edit, refactor, and deploy as you would any other.
Which LLM gives the best results?
GPT-4 and Claude give the most coherent output, but Groq's Llama 3.3 70B is much faster and usually good enough. The prompts are tuned to work well across providers.
What happens if the LLM produces broken code?
The system includes validation checks. If a syntax error is detected, you can regenerate the project with a corrective prompt.
Is my data shared with the LLM provider?
Yes — a sample of the parsed data is sent to the LLM so it can analyze the structure. Use local models if you need full privacy.
How large can the dataset be?
WebBuilder sends a sample (not the full dataset) to the LLM for analysis. The full data is loaded by the generated management command at runtime. There's no hard limit, but very large APIs may need pagination.
